# Problem Tracker

This is a competitive programming problem utility monorepo allowing users to keep track of problems they have solved,
what platform, what language, how quick, when, how often, and to get notifications of what they should work on, their activity
(daily, weekly, monthly, yearly, etc.).

## Why Build This?

Currently, I keep track of what problem's I've solved, when, and how many times. I'd like to expand this, and make it
easier for me to track, so I can keep up to date and not miss out on any categories, or forget how to solve a particular
problem.

I want to solve all problems in my list at least once a month, plus continually add more. Eventually, I plan to solve
them all, and join competitions. And hop platform to platform.

Additionally, this is an exploratory project for me to fill in some gaps in my experience, such as:

- Microservices
- Oracle DB
- UI Development, specifically try out a few different kinds, like TUI or Angular
- Kubernetes
- Cloud Providers like Azure or areas in cloud providers that I lack a lot of experience, such as GCP and AWS with EKS.
- Jenkins build system
- Testing, specifically with UI, full system, load, stress, etc.
- Hibernate

## Technology

The backend for this tooling will consist of several microservices built with Spring Boot and connected to an Oracle DB.

There may be multiple frontends for this as an exploratory project. I plan to build a TUI to manage myself, plus explore
other options such as Angular.
